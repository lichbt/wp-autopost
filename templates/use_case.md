# Use Case Template

## Structure Rules
- Name the specific use case in the first sentence — not a generic intro
- Lead with the market opportunity or problem size (a number helps)
- Show concrete implementation, not theoretical possibility
- Include a business model section (how money is made in this niche)
- End with a realistic assessment: what works well, what's challenging
