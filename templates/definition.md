# Definition / Explainer Template

Use this structure for "Definition" pillar articles.

## Article Structure

### TL;DR (2-3 sentences)
- Clear, simple definition
- Why it matters
- Who needs it

### Main Content

#### H2: What is [Topic]?
- Start with a one-sentence definition in bold
- Expand with 2-3 sentences of context
- Real-world analogy if helpful

#### H2: How [Topic] Works
- Step-by-step explanation
- Use numbered list for processes
- Include diagrams description if applicable

#### H2: Types of [Topic]
For each type:
##### H3: [Type Name]
- Definition
- Use case
- Example
- Pros and cons

#### H2: Key Features / Components
- Bullet list of main features
- Brief explanation of each
- Why each matters

#### H2: Benefits of [Topic]
- Numbered list of 5-7 benefits
- Include specific metrics when possible
- Real-world examples

#### H2: Common Use Cases
For each use case:
- **[Use Case Name]**: Description
- Who it's for
- Example scenario

#### H2: [Topic] vs Related Concepts
Compare with similar/related topics:
- How they differ
- When to use each
- Overlap areas

#### H2: Getting Started with [Topic]
- Quick start steps
- Resources for learning more
- Tools to consider

### FAQ Section (5-8 questions)
Definition-focused questions:
- "What exactly is [topic]?"
- "How does [topic] work?"
- "What are the main features of [topic]?"
- "Who should use [topic]?"
- "Is [topic] the same as [related concept]?"
- "How much does [topic] cost?"

### Meta Description
- Under 160 characters
- Start with "Learn what [topic] is..."
- Include key benefit

## Writing Style
- Define terms on first use
- Use simple language for complex concepts
- Include examples for abstract ideas
- Use analogies when helpful
- Avoid jargon without explanation

## SEO Guidelines
- Target "what is [topic]" keywords
- Include "definition", "meaning", "explained"
- Link to related terms
- Use schema markup for definitions

---

## Internal Linking (Required)

Insert **3–5 contextual internal links** within the body content using real `<a href="URL">anchor text</a>` HTML tags.

Rules:
- Use only URLs from the **INTERNAL LINK OPPORTUNITIES** list provided in the prompt
- Place links naturally inside sentences where the linked topic is directly relevant
- Use descriptive anchor text matching the target page (e.g. `<a href="...">how to monetise a dating app</a>`, not "click here")
- Spread links across different sections — do not cluster them all in one paragraph
- Never repeat the same URL twice
- Do NOT add links in the tldr, faq, or meta fields — only in the `content` field
